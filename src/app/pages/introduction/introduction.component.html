Introducing: TypeScript

What is TypeScript?

TypeScript is Javascript+.

That is to say, that valid Javascript is valid TypeScript. One of the main benefits of TypeScript is how it can improve developer experience/ergonomics, especially in a large project.

What is typing?

Typing can be described as how the type of an object (ie. string, number, boolean, etc.) is treated at compilation time and run time.

Strong vs Weak
Strongly typed languages typically define an object's type at object instantiation, whereas Weakly typed languages do not.
Static vs Dynamic
Statically typed languages typically do not allow you to reassign an object to a different type during code execution, whereas Dynamically typed languages do.
Strict
Strict languages are both Strongly and Statically typed

Javascript is both Weak and Dynamic in it's current popular version (ES202 at time of writing).

This improved experience is mainly achieved by adding strict typing to Javascript, similar to languages like Java, C#, etc.

Benefits include:

  • improved tooling, as typing removes a lot of ambiguity from the code, leaving IDEs/editors/compilers with well defined objects for code hinting, syntax coloring, etc.

  • earlier bug detection, as type bugs are not typically found until runtime, often ensuring refactoring goes smoother.

  • supported by most popular JS frameworks (and even preferred/recommended in some cases), like Vue.js, ReactJS and Angular.

  • transpiles to many versions of Javascript, allowing you to use modern language features in development and converting it at compilation to the required target for backwards compatibility (similar to Babel).

  • can be used incrementally in existing Javascript projects, allowing you to gradually transition to TypeScript as you project evolves.

  • code becomes more self-documenting, reducing developer fatigue.

  • promotes many application design best practices, including the SOLID principles.
